## 1. 背景与目标

### 1.1 背景

LeeGer 是一名室内工装设计师，日常工作涉及设计方案、客户沟通、工地巡场等多线程任务。目前缺少一个轻量的工具来管理每日待办、记录工作内容，并按公司要求输出日报。市面上的工具（滴答清单、Notion 等）功能过重或需要付费，且无法直接对接公司日报格式。

### 1.2 产品目标

1. **每日待办管理**：快速创建、完成、删除当天的工作待办事项
2. **工作记录**：记录每天实际完成的工作内容和耗时
3. **日报自动输出**：根据当天的待办和工作记录，按公司模板一键生成日报文本，可复制粘贴发送

### 1.3 成功指标

| 指标 | 目标值 | 衡量方式 |
|------|--------|----------|
| 日报生成时间 | < 30 秒 | 从点击生成到复制完成 |
| 每日使用 | 工作日每天打开 | 浏览器本地记录 |
| 数据不丢失 | 0 丢失 | 关闭浏览器后重新打开数据仍在 |

## 2. 用户与场景

### 2.1 目标用户

- **唯一用户**：LeeGer（个人使用，无需账号系统）
- **使用环境**：Windows 电脑，Chrome/Edge 浏览器
- **技术水平**：零编程经验，需要极简操作

### 2.2 使用场景

**场景 1：早晨规划**

- 触发条件：开始一天的工作
- 操作：打开 DailyFlow → 添加今天要做的待办事项
- 预期结果：清晰的任务列表，知道今天要干什么

**场景 2：工作中记录**

- 触发条件：完成一项工作后
- 操作：勾选已完成的待办 → 补充实际完成内容和备注
- 预期结果：实时记录工作进度

**场景 3：下班前写日报**

- 触发条件：一天工作结束
- 操作：点击「生成日报」→ 预览日报内容 → 复制粘贴到公司群/系统
- 预期结果：30 秒内完成日报输出，格式符合公司要求

### 3.2 核心功能详述

#### 功能 1：每日待办管理

**描述：** 用户可以为每一天创建待办事项，标记完成状态，删除不需要的条目。

**用户故事：** 作为 LeeGer，我希望快速添加今天的待办事项，以便清楚知道今天要做什么。

**验收标准：**

- [ ] 可以输入文字并添加为待办事项
- [ ] 点击复选框可标记为已完成（有视觉区分）
- [ ] 可以删除单个待办
- [ ] 待办按日期隔离，不同日期的互不影响
- [ ] 刷新页面后数据仍在

**交互说明：**

- 输入框 + 回车/按钮 添加待办
- 待办项显示：复选框 + 文本 + 删除按钮
- 已完成项有删除线或灰色样式

### 3.3 数据模型

#### 3.3.1 核心数据实体

| 实体 | 字段 | 类型 | 说明 |
|------|------|------|------|
| Todo | id | string | UUID唯一标识 |
| Todo | text | string | 待办内容 |
| Todo | completed | boolean | 完成状态 |
| Todo | project | string | 项目标签 |
| Todo | priority | string | 优先级：urgent/important/normal |
| Todo | note | string | 备注内容 |
| Todo | createdAt | datetime | 创建时间 |
| Todo | completedAt | datetime | 完成时间 |
| Project | id | string | UUID唯一标识 |
| Project | name | string | 项目名称 |
| Template | id | string | UUID唯一标识 |
| Template | name | string | 模板名称 |
| Template | text | string | 模板内容 |
| TimelineEvent | id | string | UUID唯一标识 |
| TimelineEvent | type | string | 操作类型：add/complete/edit |
| TimelineEvent | todoId | string | 关联待办ID |
| TimelineEvent | timestamp | datetime | 操作时间 |
| TimelineEvent | description | string | 操作描述 |

#### 3.3.2 数据关系

```
Todo (1) ──── (1) Project
Todo (1) ──── (N) TimelineEvent
```

#### 3.3.3 状态流转

**待办状态：**
```
[未完成] ──勾选──> [已完成] ──取消勾选──> [未完成]
```

**日报生成流程：**
```
点击生成 → 检查已完成待办 → 无：提示用户
                           → 有：拼装日报文本 → 弹出预览 → 复制到剪贴板
```

### 3.4 业务规则
