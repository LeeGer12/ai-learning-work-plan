## 1. 背景与目标

### 1.1 背景

LeeGer 是一名室内工装设计师，日常工作涉及设计方案、客户沟通、工地巡场等多线程任务。目前缺少一个轻量的工具来管理每日待办、记录工作内容，并按公司要求输出日报。市面上的工具（滴答清单、Notion 等）功能过重或需要付费，且无法直接对接公司日报格式。

### 1.2 产品目标

1. **每日待办管理**：快速创建、完成、删除当天的工作待办事项
2. **工作记录**：记录每天实际完成的工作内容和耗时
3. **日报自动输出**：根据当天的待办和工作记录，按公司模板一键生成日报文本，可复制粘贴发送

### 1.3 成功指标

| 指标 | 目标值 | 衡量方式 |
|------|--------|----------|
| 日报生成时间 | < 30 秒 | 从点击生成到复制完成 |
| 每日使用 | 工作日每天打开 | 浏览器本地记录 |
| 数据不丢失 | 0 丢失 | 关闭浏览器后重新打开数据仍在 |

## 2. 用户与场景

### 2.1 目标用户

- **唯一用户**：LeeGer（个人使用，无需账号系统）
- **使用环境**：Windows 电脑，Chrome/Edge 浏览器
- **技术水平**：零编程经验，需要极简操作

### 2.2 使用场景

**场景 1：早晨规划**

- 触发条件：开始一天的工作
- 操作：打开 DailyFlow → 添加今天要做的待办事项
- 预期结果：清晰的任务列表，知道今天要干什么

**场景 2：工作中记录**

- 触发条件：完成一项工作后
- 操作：勾选已完成的待办 → 补充实际完成内容和备注
- 预期结果：实时记录工作进度

**场景 3：下班前写日报**

- 触发条件：一天工作结束
- 操作：点击「生成日报」→ 预览日报内容 → 复制粘贴到公司群/系统
- 预期结果：30 秒内完成日报输出，格式符合公司要求

### 3.3 数据模型

#### 3.3.1 核心数据实体

| 实体 | 字段 | 类型 | 说明 |
|------|------|------|------|
| Todo | id | string | UUID唯一标识 |
| Todo | text | string | 待办内容 |
| Todo | completed | boolean | 完成状态 |
| Todo | project | string | 项目标签 |
| Todo | priority | string | 优先级：urgent/important/normal |
| Todo | note | string | 备注内容 |
| Todo | createdAt | datetime | 创建时间 |
| Todo | completedAt | datetime | 完成时间 |
| Todo | carriedFrom | date/string | 顺延来源日期，可为空 |
| Todo | sourceTodoId | string | 顺延来源待办ID，可为空 |
| Todo | carriedAt | datetime | 顺延创建时间，可为空 |
| Project | id | string | UUID唯一标识 |
| Project | name | string | 项目名称 |
| Template | id | string | UUID唯一标识 |
| Template | name | string | 模板名称 |
| Template | text | string | 模板内容 |
| TimelineEvent | id | string | UUID唯一标识 |
| TimelineEvent | type | string | 操作类型：add/complete/edit/delete/carry |

| TimelineEvent | timestamp | datetime | 操作时间 |
| TimelineEvent | description | string | 操作描述 |

#### 3.3.2 数据关系

```
Todo (1) ──── (1) Project
Todo (1) ──── (N) TimelineEvent
```

#### 3.3.3 状态流转

**待办状态：**
```
[未完成] ──勾选──> [已完成] ──取消勾选──> [未完成]
```

**日报生成流程：**
```
点击生成 → 检查已完成待办 → 无：提示用户
                           → 有：拼装日报文本 → 弹出预览 → 复制到剪贴板
```

### 3.4 业务规则

#### 3.4.1 待办管理规则

| 规则 | 说明 |
|------|------|
| 日期隔离 | 待办按日期存储，不同日期互不影响 |
| 优先级排序 | 紧急 > 重要 > 普通，同优先级按创建时间 |
| 自动保存 | 待办变更后自动保存到localStorage |
| 删除确认 | 删除待办需二次确认（或提供撤销功能） |

#### 3.4.2 日报生成规则

| 规则 | 说明 |
|------|------|
| 数据来源 | 仅包含当天已完成的待办 |
| 输出格式 | 日期 + 编号列表 |
| 空数据处理 | 无已完成待办时提示用户 |
| 编辑支持 | 生成后支持手动编辑再复制 |

#### 3.4.3 日历规则

| 规则 | 说明 |
|------|------|
| 有记录标记 | 当天有待办（无论是否完成）显示表情 |
| 今天高亮 | 今天日期使用高亮圆形和表情 |

## 4. 视觉设计规范

> v1.14 起采用暖纸张质感的工作台风格：低饱和背景、白色卡片、陶土色主操作、细腻阴影和轻量动效。视觉实现仍保持单 HTML、零外部依赖。

### 4.1 配色方案

| 角色 | 色值 / 变量 | 用途 |
|------|-------------|------|
| 主文字 | `--ink: #2C2825` | 标题、待办正文、关键数字 |
| 次级文字 | `--ink-light: #5C5652` | 按钮文字、正文说明 |
| 辅助文字 | `--ink-muted: #9A938C` | 时间、占位符、统计标签 |
| 页面背景 | `--paper: #F5F1EC` | 整体画布，叠加轻微纸张纹理 |
| 卡片背景 | `--paper-card: #FDFBF8` | 日历、待办、统计、弹窗 |
| 暖灰背景 | `--paper-warm: #EDE8E1` | hover、输入框、局部底色 |
| 主操作色 | `--terracotta: #C4704B` | 主按钮、复选框选中、紧急/顺延标签 |
| 辅助强调 | `--olive: #7A8B6F` | 重要标签、辅助状态 |
| 项目强调 | `--clay: #D4A574` | 项目标签、柔和提示 |
| 分割线 | `--border: #E5DED5` | 卡片边框、输入框、分割线 |

### 4.2 字体层级

页面使用本地可用字体栈：标题优先 `Newsreader, Georgia, serif`，正文优先 `DM Sans, -apple-system, "Microsoft YaHei", sans-serif`。未安装前两者时自动回退系统字体，不产生网络依赖。

| 层级 | 字号 | 字重 | 用途 |
|------|------|------|------|
| Logo/日期 | 19px ~ 22px | 600 | 顶部品牌、当前日期 |
| 模块标题 | 15px ~ 17px | 600 | 日历、今日待办、统计、时间轴 |
| 待办文本 | 14px | 400 | 待办事项内容 |
| 按钮文字 | 13px ~ 14px | 500 | 主按钮、次按钮、模板按钮 |
| 辅助信息 | 10px ~ 12px | 400/500 | 标签、备注、时间、统计说明 |

### 4.3 间距规范

基于 `8px` 网格系统：

| 场景 | 间距 |
|------|------|
| 顶部栏高度 | 60px |
| 页面内容区内边距 | 28px |
| 三栏之间间距 | 24px |
| 卡片之间间距 | 16px |
| 卡片内部内边距 | 16px ~ 24px |
| 列表项间距 | 8px ~ 16px |
| 图标与文字间距 | 6px ~ 10px |

### 4.4 组件样式

**卡片：**
- 背景：`--paper-card`
- 圆角：`16px ~ 20px`
- 边框：`1px solid --border-light`
- 阴影：轻量多层阴影，强调白色工作台质感
- 装饰：卡片顶端可用细渐变线增强层级，不使用外部图片

**按钮：**
- 主按钮使用陶土色渐变，hover 轻微上浮
- 次按钮使用半透明白底和浅边框
- 顶部快捷模板、日期按钮、管理按钮保持 6px ~ 10px 圆角

**待办标签：**
- 项目标签：黏土色系
- 紧急标签：陶土色系
- 重要标签：橄榄色系
- 顺延标签：陶土浅底，文本为「顺延自 X月X日」

**模态弹窗：**
- 遮罩固定全屏居中，`.modal-overlay.active` 使用 flex 居中
- 弹窗最大宽度约 `560px`，圆角 `16px`，强阴影
- 日报预览和备注输入保持可编辑、可复制

### 4.5 整体布局

```
┌──────────────────────────────────────────────────────────────────┐
│  顶部栏：Logo + 搜索框 + 快捷模板按钮                   60px    │
├────────────┬─────────────────────────────┬───────────────────────┤
│            │                             │                       │
│  日历卡片   │   今日待办 / 搜索结果         │  ⚡ 快捷操作            │
│            │                             │  [生成日报]            │
│ ◀ 月份 ▶    │  ☐ 待办事项 [项目] [紧急]     │  [顺延到明日]          │
│ 一 二 三... │  ☐ 顺延事项 [顺延自 6月24日] │  [导出备份]            │
│            │  ☑ 已完成事项                │                       │
│            │                             │  📊 今日统计           │
│            │  [+ 添加待办]                │  📈 本周概览           │
│            │                             │  🕐 工作时间轴         │
└────────────┴─────────────────────────────┴───────────────────────┘
```

- **布局模式**：三栏布局，日历约 `280px`，主区自适应，侧栏约 `280px`
- **定位**：顶部栏 sticky，左右侧栏在桌面宽度下 sticky
- **响应式**：优先桌面端，窄屏下改为单列堆叠
- **视觉原则**：保持高密度信息面板，不做营销式落地页
