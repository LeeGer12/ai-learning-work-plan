# GitHub 上传指南

## 前置条件

- 已安装 Git（`git --version` 检查）
- 已安装 GitHub CLI（`gh --version` 检查）
- 有 GitHub 账号

---

## 方式一：GitHub CLI（推荐）

### 1. 登录认证

**浏览器方式**（网络好时用）：
```bash
gh auth login -h github.com
# 选择：GitHub.com → HTTPS → 用浏览器登录
```

**Token 方式**（网络不稳定时用）：
```bash
# 先去 https://github.com/settings/tokens 生成 token
# 勾选：repo、admin:org
gh auth login -h github.com --with-token <<< "ghp_你的token"
```

### 2. 初始化本地仓库

```bash
cd "你的项目目录"
git init
```

### 3. 创建 .gitignore

```gitignore
# 依赖
node_modules/

# 临时文件
*.tmp
*.log

# 系统文件
.DS_Store
Thumbs.db

# 编辑器配置
.vscode/
.idea/
```

### 4. 提交代码

```bash
git add .
git commit -m "init: 项目初始化"
```

### 5. 创建远程仓库并推送

```bash
gh repo create 仓库名 --public --source=. --push --description "项目描述"
```

**参数说明**：
- `--public`：公开仓库（私有仓库用 `--private`）
- `--source=.`：当前目录作为源
- `--push`：创建后自动推送
- `--description`：仓库描述

**完成后会输出仓库地址**：`https://github.com/用户名/仓库名`

---

## 方式二：手动操作

### 1. 在 GitHub 网页创建仓库

1. 打开 https://github.com/new
2. 填写仓库名
3. 选择 Public/Private
4. **不要勾选** README、.gitignore、License（本地已有）
5. 点击 Create repository

### 2. 关联远程仓库并推送

```bash
cd "你的项目目录"
git init
git add .
git commit -m "init: 项目初始化"
git remote add origin https://github.com/用户名/仓库名.git
git push -u origin master
```

## 常见问题

### Q: `gh auth login` 超时/连不上？

**原因**：国内网络访问 GitHub 不稳定

**解决**：用 Token 方式登录（见上方步骤一）

### Q: Token 权限不足？

**错误信息**：`missing required scope 'read:org'`

**解决**：重新生成 Token，勾选：
- ✅ repo（全部）
- ✅ admin:org

### Q: `git push` 要求输入密码？

**原因**：未配置凭证

**解决**：
```bash
# 用 gh 配置 git 凭证
gh auth setup-git
```

### Q: 提交时提示 `LF will be replaced by CRLF`？

**原因**：Windows 换行符问题

**解决**：正常现象，不影响使用，忽略即可

## 安全提醒

⚠️ **Token 不要泄露**：
- 不要把 Token 写进代码或文档
- 不要在聊天中分享 Token
- 如果泄露了，立即去 https://github.com/settings/tokens 删除
